# Plan: L-Shaped Routing + Haptics Only at Turns

## Summary
- **Routing:** L-shaped path with single corner at optimal point
- **Haptics:** ONLY at corner turns, NOT every intermediate waypoint

---

## Implementation

### 1. L-Shaped Routing (Smart Corner)
File: `lib/data/models/corridor_map.dart`

Keep existing `CorridorRouter` algorithm:
- Option 1: corner at (end.x, start.y) - go horizontal first
- Option 2: corner at (start.x, end.y) - go vertical first
- Selects whichever gives shorter total path

**Important:** The path should be CLEAN L-shape:
- Start → Corner → Destination (only 3 points)
- NO intermediate waypoints between corners
- Visualization shows straight lines between these points only

### 2. Mark Corner Waypoints
File: `lib/data/models/corridor_map.dart`

Return path with corner information:
```dart
class PathCorner {
  final Offset position;
  final bool isCorner;
  final TurnDirection? turn; // null for intermediate, L/R for corner
}

List<PathCorner> findPathWithCorners(Offset start, Offset end)
```

### 3. Only Trigger Haptics at Corners
File: `lib/presentation/pages/uwb_map/uwb_map_page.dart`

Modify `_triggerHaptic` to check if current waypoint is a corner:
- `isCorner == true` → trigger Turn Left/Right haptic
- `isCorner == false` → NO haptic (just moving forward)
- At destination → trigger Arrived haptic

### 4. Visual L-Shape (Important)
In map painter (`_MapPainter`):
- Draw route as TWO straight lines:
  - Line 1: Start → Corner (90° turn point)
  - Line 2: Corner → Destination
- Mark corner with special indicator (diamond/square)
- NO intermediate dots between start-corner-dest
- Route should look like a clear "L" shape on screen

**Current bug to fix:** Currently shows many small dots/wide line segments. Should show clean L.

### 5. Real-Time Route Update (NEW)
File: `lib/presentation/pages/uwb_map/uwb_map_page.dart`

Currently the route is calculated ONCE when destination is tapped. User wants route to update in REAL-TIME as they move.

**Implementation:**
- Listen to UWB position updates in real-time
- When user gets close to current waypoint (within `_waypointRadius`), automatically:
  1. Recalculate route from new position to destination
  2. Update visualization with new path
  3. Update corner waypoints

**Code to add in `_UwbMapPageState`:**
```dart
// In position stream listener:
_posSub = _uwbService.positionStream.listen((pos) {
  if (_path != null) {
    final dist = _distToCurrentWaypoint;
    if (dist != null && dist < _waypointRadius) {
      // User reached waypoint - recalculate from new position to destination
      final newPath = _NavPath.build(
        Offset(pos.x, pos.y),
        _path!.destination,
      );
      setState(() => _path = newPath);
    }
  }
});
```

## Expected Behavior

**Route:** Start (0,0) → Corner (4,0) → Destination (4,7)
- Waypoints: start → corner → dest
- Haptics: "Turn Right" at corner → "Arrived" at destination
- Only 2 vibrations TOTAL

**Real-time update:**
- User walks toward corner
- When within 0.6m of corner, route recalculates from new position
- New L-shape drawn from where user is now

**Current Behavior to Keep:**

- If going both x and y: L-shaped via optimal corner
