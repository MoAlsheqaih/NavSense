# NavSense Simulation Framework Plan

## Goal
Create a complete simulation framework to test the indoor navigation system without hardware (BLE/UWB). The simulation will cover the full Input → Process → Output flow.

## Correction: User Clarified Requirements

The user clarified that **off-route handling is NOT needed**. The system should simply:
1. **INPUT**: Random position (x, y) on floor (simulating BLE/UWB)
2. **PROCESS**: Generate route from current position → destination (Dijkstra)
3. **OUTPUT**: Instructions (Turn Left, Turn Right, Go Straight)

No off-route detection, no drift simulation - just continuous instruction generation.

### How It Works

1. **Initial State**: Map shows floor grid, tap to place destination (green dot)
2. **Set Origin**: Customer position starts at (0, 0) or tap to set
3. **Generate Route**: Dijkstra calculates path from origin → destination
4. **Move Customer**: Drag blue dot - route recalculates, instructions update
5. **Instructions Panel**: Shows current instruction (Turn L/R/Straight) + distance

### Key Features (Confirmed by User)

- **Free drag**: Customer dot can be anywhere on floor (continuous)
- **Real-time updates**: Instructions update instantly while dragging
- **Tap to set**: First tap = origin (blue), second tap = destination (green)
- **Route recalculation**: New route generated on position change
- **Visual feedback**: Route path drawn as dashed line, waypoints marked
- **No hardware**: Pure simulation - works on iOS/Android simulator
