# Plan: Fix Slow Routing Computation

## Problem
Route computation feels slow because there's an artificial delay built into the system.

## Root Cause
In `lib/core/constants/app_constants.dart`:
```dart
static const Duration mockRouteDelay = Duration(milliseconds: 800);
```

This delay is called in both:
- `lib/data/datasources/floor_route_datasource.dart` (line 425)
- `lib/data/datasources/mock_route_datasource.dart` (line 19)

The actual Dijkstra algorithm is very fast (~1450 nodes). The 800ms delay was added as a simulated "computation time" for testing but should be removed for instant routing.

## Solution
Reduce `mockRouteDelay` to a small value (e.g., 50ms) to maintain subtle UI feedback, or set to `Duration.zero` for instant computation.

## Implementation
Edit `lib/core/constants/app_constants.dart`:
- Change `mockRouteDelay` from `Duration(milliseconds: 800)` to `Duration(milliseconds: 50)` (or `Duration.zero`)

This change will make route computation nearly instant.
