# Plan: Integrating Python UWB BLE Gateway with NavSense Flutter App

## Summary

**User Requirements:**
- Use Python as BLE-to-UDP gateway (recommended)
- Replace Tkinter GUI with Flask web interface
- Data format: `{"links": [{"A": "addr", "R": distance}, ...]}`

The Python script will become a headless gateway that:
1. Connects to ESP32 UWB wearable via BLE (Nordic UART Service)
2. Receives raw UWB range data from 3 anchors
3. Performs 2D trilateration to calculate tag position
4. Sends position data to Flutter via UDP
5. Receives haptic commands from Flutter via UDP and forwards to ESP32 via BLE

---

## Architecture

```
ESP32 UWB Wearable ←──── BLE (NUS) ────→ Python Gateway (Flask) ←─── UDP ────→ Flutter App
                         (ranges)                                  (x,y positions)
                                                                          
Flutter App ────────────── UDP ──────────→ Python Gateway ←──── BLE (NUS) ──── ESP32
   (haptic: F/B/L/R)                           (forward cmd)                     (haptic feedback)
```

---

## Implementation Steps

### Step 1: Create Flask Web Server in Python

Replace Tkinter GUI with Flask:
- Keep BLE connection logic (bleak)
- Keep trilateration math
- Add Flask routes for:
  - `GET /` - Web dashboard showing position and controls
  - `GET /position` - JSON endpoint returning current (x,y)
  - `POST /command` - Receive haptic commands from web/Flutter

### Step 2: Add UDP Communication

- Listen on port 8081 for position broadcasts to Flutter
- Listen on port 8082 for commands from Flutter
- Forward BLE data to Flask endpoints

### Step 3: Update Flutter RealUwbService

- Already listens on port 8080 - verify JSON format compatibility
- Update if needed to match Python output format

### Step 4: Add Flutter Haptic Gateway

- Create `HapticUdpService` to send commands to Python
- Send "F", "B", "L", "R" to port 8082
