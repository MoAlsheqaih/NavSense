# UWB Navigation & Haptic Integration Fix Plan

## Objective
Fix two critical issues:
1. **Poor routing/direction**: Navigation calculations don't match physical layout
2. **Haptic not reaching motors**: Commands not sent via BLE to ESP32 motors

---

## Issue 1: Routing/Direction Problem

### Current Analysis
The Python gateway uses raw angle-based direction calculation:
- Assumes 0° = positive X axis, 90° = positive Y
- 4 quadrant model: R/F/L/B
- Doesn't account for user's physical orientation

### Problems Identified
1. **Static orientation assumption**: Assumes user faces "Forward" (positive Y)
2. **No smoothing**: Direction changes instantly without filtering
3. **No heading fusion**: Doesn't account for device compass/heading
4. **Binary F/L/R/B**: No diagonal or intermediate directions

### Proposed Fixes
1. Add direction smoothing/hysteresis (don't change direction on small movements)
2. Add configurable direction mapping based on actual room layout
3. Improve threshold settings for direction changes
4. Consider adding compass heading for user-oriented directions

## Issue 2: Haptic Commands Not Reaching ESP32 Motors

### Current Architecture (Problematic)
```
Flutter App → UDP:8082 → Python Gateway → BLE → ESP32 → Motors
                         ↑
                    This link is broken!
```

### Current Implementation
- Flutter uses `UdpWearableHapticService` sending to UDP port 8082
- Python gateway receives commands but may not forward correctly
- BLE connection to ESP32 exists but haptic write may fail

### Root Causes
1. **UDP Socket Issue**: `UdpWearableHapticService` sends to `loopbackIPv4` (127.0.0.1) - this won't work on physical device!
2. **Python Gateway**: May not be forwarding UDP commands to BLE properly
3. **BLE Write**: ESP32 characteristic write callback may not trigger haptic

### Proposed Fixes
Option A: **Direct BLE from Flutter to ESP32**
- Add native BLE haptic service in Flutter using flutter_blue_plus
- Connect directly to ESP32's NUS RX characteristic
- This bypasses Python gateway entirely

Option B: **Fix UDP→Python→BLE chain**
- Fix UDP destination (use actual gateway IP, not loopback)
- Ensure Python forwards to BLE correctly
- Debug ESP32 RX callback

// New file: lib/services/haptic/ble_wearable_haptic_service.dart

import 'package:flutter_blue_plus/flutter_blue_plus.dart';
import 'package:navsense/domain/entities/route_plan.dart';
import 'package:navsense/services/haptic/wearable_haptic_service.dart';
import 'package:navsense/services/haptic/haptic_pattern.dart';

class BleWearableHapticService implements WearableHapticService {
  static final String SERVICE_UUID = "6E400001-B5A3-F393-E0A9-E50E24DCCA9E";
  static final String RX_CHAR_UUID = "6E400002-B5A3-F393-E0A9-E50E24DCCA9E";
  
  BluetoothDevice? _device;
  BluetoothCharacteristic? _rxChar;
  
  @override
  bool get isConnected => _device?.state == BluetoothDeviceState.connected;
  
  @override
  Future<void> connect() async {
    // Scan for ESP32 with name "UWB-Wearable"
    final devices = await FlutterBluePlus.scan(timeout: const Duration(seconds: 10));
    final esp32 = devices.firstWhere(
      (d) => d.platformName == "UWB-Wearable",
      orElse: () => throw Exception("ESP32 not found"),
    );
    
    await esp32.connect();
    _device = esp32;
    
    // Get NUS service and RX characteristic
    final services = await esp32.discoverServices();
    final nusService = services.firstWhere((s) => s.uuid.str == SERVICE_UUID);
    _rxService = nusService.characteristics.firstWhere((c) => c.uuid.str == RX_CHAR_UUID);
  }
  
  @override
  Future<void> triggerDirection(TurnDirection direction) async {
    if (_rxChar == null) return;
    
    String cmd;
    switch (direction) {
      case TurnDirection.left: cmd = 'L'; break;
      case TurnDirection.right: cmd = 'R'; break;
      case TurnDirection.straight: cmd = 'F'; break;
      case TurnDirection.arrived: return;
    }
    
    await _rxChar?.write(cmd.codeUnits);
  }
}
```

## Implementation Plan (Original - ESP32 Haptic)

### 1. Non-Blocking State Machine

Replace the blocking `delay()` with a **finite state machine (FSM)** using `millis()`:

The original pattern sequences: DOWN → CENTER → TARGET for each rep (3 reps total = 9 pulses)

```
States: IDLE → PULSE → GAP → (track which of 3 pulses in sequence)
```

**New Variables:**
```c++
enum HapticState { HAPTIC_IDLE, HAPTIC_PULSE, HAPTIC_GAP };

// Pulse sequence tracking: 0=DOWN, 1=CENTER, 2=TARGET
uint8_t pulseIndex = 0;  // Which pulse in sequence (0,1,2)
uint8_t currentRep = 0;  // Current repetition (0-2)

HapticState hapticState = HAPTIC_IDLE;
uint8_t currentTarget = 0;
unsigned long stateStartTime = 0;

// For interrupt handling
char pendingCommand = 0;
bool commandPending = false;
```

**FSM Update Function:**
```c++
void updateHaptic() {
    // Check for new command interrupts
    if (commandPending && hapticState != HAPTIC_PULSE) {
        startHapticPattern(pendingCommand, PRIORITY_NORMAL);
        commandPending = false;
    }

    if (hapticState == HAPTIC_IDLE) return;

    unsigned long elapsed = millis() - stateStartTime;

    switch (hapticState) {
        case HAPTIC_PULSE:
            if (elapsed >= PULSE_DURATION_MS) {
                allOff();  // Turn off current motor
                stateStartTime = millis();
                hapticState = HAPTIC_GAP;
            }
            break;

        case HAPTIC_GAP: {
            uint16_t gapTime = (pulseIndex < 2) ? INTER_PULSE_MS : REP_INTERVAL_MS;

            if (elapsed >= gapTime) {
                advancePulseSequence();
            }
            break;
        }

        default:
            break;
    }
}

// Handle the DOWN → CENTER → TARGET sequence
void advancePulseSequence() {
    pulseIndex++;

    if (pulseIndex > 2) {  // Completed sequence (0,1,2)
        pulseIndex = 0;
        currentRep++;

        if (currentRep >= REPS) {
            // All repetitions complete
            hapticState = HAPTIC_IDLE;
            pulseIndex = 0;
            currentRep = 0;
            currentTarget = 0;
            Serial.println("[HAPTIC] Pattern complete");
            return;
        }
    }

    // Determine which motor to pulse
    uint8_t motor;
    if (pulseIndex == 0) motor = PIN_DOWN;      // First: DOWN
    else if (pulseIndex == 1) motor = PIN_CENTER;  // Second: CENTER
    else motor = currentTarget;                 // Third: TARGET direction

    analogWrite(motor, getIntensityForMotor(motor));
    hapticState = HAPTIC_PULSE;
    stateStartTime = millis();
}

void startHapticPattern(char direction, CommandPriority priority) {
    currentTarget = getTargetPin(direction);
    currentRep = 0;
    pulseIndex = 0;

    // Start with DOWN pulse
    analogWrite(PIN_DOWN, getIntensityForMotor(PIN_DOWN));
    hapticState = HAPTIC_PULSE;
    stateStartTime = millis();

    Serial.print("[HAPTIC] Starting: ");
    Serial.println(direction);
}

uint8_t getIntensityForMotor(uint8_t motor) {
    if (motor == PIN_UP)     return 255;  // Full power - primary forward
    if (motor == PIN_DOWN)   return 180;  // Reduced - anchor point
    if (motor == PIN_LEFT)   return 220;  // Medium-high - left

    return 200;  // Default
}

### 4. Safety Timeout

Add watchdog-style timeout to prevent stuck motors:

```c++
const unsigned long HAPTIC_SAFETY_TIMEOUT_MS = 5000;  // 5 seconds max

unsigned long hapticStartTime = 0;

void startHaptic(unsigned long now) {
    hapticStartTime = now;
}

void checkHapticSafety() {
    if (hapticState != HAPTIC_IDLE && (millis() - hapticStartTime) > HAPTIC_SAFETY_TIMEOUT_MS) {
        allOff();
        hapticState = HAPTIC_IDLE;
        Serial.println("[HAPTIC] Safety timeout triggered");
    }
}
```

### 5. Motor Pin Management Fix

Ensure proper pin cleanup to prevent ghost vibrations:

```c++
void allOff() {
    // Set all pins to 0 before changing mode to prevent floating
    analogWrite(PIN_UP,     0);
    analogWrite(PIN_LEFT,   0);
    analogWrite(PIN_CENTER, 0);
    analogWrite(PIN_RIGHT,  0);
    analogWrite(PIN_DOWN,   0);

    // Brief digital LOW to ensure complete discharge
    digitalWrite(PIN_UP,     LOW);
    digitalWrite(PIN_LEFT,   LOW);
    digitalWrite(PIN_CENTER, LOW);
    digitalWrite(PIN_RIGHT,  LOW);
    digitalWrite(PIN_DOWN,   LOW);
}
